# Service-Model Refactor Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the overloaded `CollectionSource` enum with two orthogonal, open-ended fields (`Collection.service: String?`, `Order.marketplaceService: String?`) plus an SDK service registry, so adding a new service is a registry entry, not a schema migration.

**Architecture:** Four sequential phases, each independently shippable and revertible (per `2026-05-15-asset-service-model.md` §9). 2a adds the new columns + dual-writes them in the indexer (old `source` keeps being written). 2b ships the SDK registry + `service` field. 2c cuts frontends over to `getService()`. 2d (after a production soak) stops dual-writing and drops the old enum. Authority for every decision: the locked architecture docs `00`–`09`, especially `05-service-model.md`.

**Tech Stack:** Bun (`~/.bun/bin/bun`, never node/npm), Prisma v5 + PostgreSQL, Hono, `@medialane/sdk` (tsup, TypeScript), Next.js (dapp + io).

**Verification discipline (skill-TDD adapted per CLAUDE.md — user instruction overrides skill default):** This codebase has **no test runner** (backend/SDK/dapp confirmed; io has `typecheck`). Per `medialane-backend/CLAUDE.md` ("No linting or test runner configured. Verify with curl against localhost:3000") and memory `feedback_run_build_before_push`, every "verify" step uses **`tsc --noEmit`**, **`psql` assertions**, **`curl` against `localhost:3000`**, and **full `bun run build`** — not a fabricated test harness. These are the project's real correctness gates; treat a failing one exactly as you would a failing unit test (stop, fix, re-run).

**Canonical reconciliations applied (do not deviate):**
- Service IDs are the **long forms** from `01-core-model.md` §III: `mip-erc721`, `mip-erc1155`, `ip-erc721`, `pop-protocol`, `drop-collection`, `medialane-marketplace-erc721`, `medialane-marketplace-erc1155`. The 2026-05-15 draft's short forms (`pop`, `drop`) are **superseded**.
- License default is **`CC BY-SA`** (`04-licensing-model.md` §III), not the draft's `CC-BY-4.0`.
- `marketplaceContract` column is **kept** (draft §10.1, locked) as an explorer-link helper — never dropped, never a behavior discriminator.
- **No `ERC20` enum value** is added in this refactor (YAGNI / `09 §VI` — no active service uses it; it ships with the first ERC-20 service).

### Task 2A.3: Backfill script for existing rows

**Files:**

- [ ] **Step 1: Write the backfill script**

Create `src/scripts/backfill-service-id.ts`:

```ts
/**
 * One-time, idempotent backfill: derive Collection.service from legacy
 * Collection.source, and Order.marketplaceService from Order.marketplaceContract.
 * Leaves legacy columns untouched. Safe to re-run. --dry-run to preview.
 *
 * Run: ~/.bun/bin/bun run backfill-service-id [--dry-run]
 */
import { PrismaClient } from "@prisma/client";
import { normalizeAddress } from "../utils/starknet.js";
import {
  MARKETPLACE_CONTRACT_MAINNET,
  MARKETPLACE_1155_CONTRACT_MAINNET,
} from "../config/constants.js";
import { createLogger } from "../utils/logger.js";

const log = createLogger("backfill-service-id");
const prisma = new PrismaClient();
const DRY = process.argv.includes("--dry-run");

// Legacy source → canonical long-form service ID (01-core-model §III).
// null = no Medialane service (external).
const SOURCE_TO_SERVICE: Record<string, string | null> = {
  MEDIALANE_ERC721: "mip-erc721",
  MEDIALANE_REGISTRY: "mip-erc721", // legacy alias of the ERC-721 registry
  MEDIALANE_ERC1155: "mip-erc1155",
  ERC1155_FACTORY: "mip-erc1155", // legacy alias
  POP_PROTOCOL: "pop-protocol",
  COLLECTION_DROP: "drop-collection",
  EXTERNAL: null,
  EXTERNAL_ERC721: null,
  EXTERNAL_ERC1155: null,
  PARTNERSHIP: null,
  IP_TICKET: null,
  IP_CLUB: null,
  GAME: null,
};

async function backfillCollections() {
  const rows = await prisma.collection.findMany({
    where: { service: null },
    select: { id: true, source: true },
  });
  let updated = 0;
  for (const r of rows) {
    if (!(r.source in SOURCE_TO_SERVICE)) {
      log.warn({ id: r.id, source: r.source }, "unmapped source — skipping");
      continue;
    }
    const service = SOURCE_TO_SERVICE[r.source];
    if (service === null) continue; // external: leave service null
    log.info({ id: r.id, source: r.source, service }, DRY ? "[dry] would set" : "set");
    if (!DRY) {
      await prisma.collection.update({ where: { id: r.id }, data: { service } });
    }
    updated++;
  }
  return { scanned: rows.length, updated };
}

async function backfillOrders() {
  const MARKETPLACE_TO_SERVICE: Record<string, string> = {
    [normalizeAddress(MARKETPLACE_CONTRACT_MAINNET)]: "medialane-marketplace-erc721",
    [normalizeAddress(MARKETPLACE_1155_CONTRACT_MAINNET)]: "medialane-marketplace-erc1155",
  };
  const rows = await prisma.order.findMany({
    where: { marketplaceService: null, marketplaceContract: { not: null } },
    select: { id: true, marketplaceContract: true },
  });
  let updated = 0;
  for (const r of rows) {
    const key = normalizeAddress(r.marketplaceContract as string);
    const svc = MARKETPLACE_TO_SERVICE[key];
    if (!svc) {
      log.warn({ id: r.id, marketplaceContract: r.marketplaceContract }, "unknown marketplace — skipping");
      continue;
    }
    log.info({ id: r.id, svc }, DRY ? "[dry] would set" : "set");
    if (!DRY) {
      await prisma.order.update({ where: { id: r.id }, data: { marketplaceService: svc } });
    }
    updated++;
  }
  return { scanned: rows.length, updated };
}

async function main() {
  log.info({ dryRun: DRY }, "backfill-service-id start");
  const c = await backfillCollections();
  const o = await backfillOrders();
  log.info({ collections: c, orders: o }, "backfill-service-id done");
  await prisma.$disconnect();
}

main().catch(async (e) => {
  log.error({ err: e }, "backfill failed");
  await prisma.$disconnect();
  process.exit(1);
});
```
